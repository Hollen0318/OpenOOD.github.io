A standardized benchmark for Out-of-Distribution Detection

The goal of Out-Of-Distribution detection is to identify and handle data samples that are not from the same distribution as the training data, also known as in-distribution (ID) data. These OOD samples might come from unknown or novel classes, or they might be instances of known classes but with different characteristics due to variations in data collection or preprocessing.

from OpenOOD.utils import load_model
# Load a model from the Open Models
model = load_model(model_name='Rebuffi2021Fixing_70_16_cutmix_extra',
                   dataset='cifar10',
                   threat_model='Linf')

# Evaluate the OOD of the model using AutoAttack
from OpenOOD.eval import benchmark
clean_acc, near_ood_auroc, far_ood_auroc = benchmark(model,
                                  dataset='cifar10',
                                  threat_model='Linf')

FAQ

➤ How does the OpenOOD leaderboard differ from the AutoAttack leaderboard? ü§î
The AutoAttack leaderboard was the starting point of OpenOOD. Now only the OpenOOD leaderboard is actively maintained.

➤ How is it related to libraries like foolbox / cleverhans / advertorch? ü§î
These libraries provide implementations of different attacks. Besides the standardized benchmark, OpenOOD additionally provides a repository of the most accurate models. So you can start using the OOD models in one line of code (see the tutorial here).
